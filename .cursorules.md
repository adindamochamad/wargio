## 2. Problem Statement (untuk semua narasi kode & komentar)

64 juta UMKM dan warung di Indonesia mengelola bisnis secara manual: buku catatan hutang, ingatan stok, intuisi harga. Mereka tidak punya akses ke sistem manajemen bisnis karena terlalu mahal, terlalu kompleks, dan tidak dalam Bahasa Indonesia.

**Wargio** adalah AI agent yang bekerja seperti asisten pintar di balik meja warung — bisa ditanya dalam bahasa natural ("Bu, mie instan tinggal berapa?", "Siapa yang belum bayar hutang minggu ini?", "Produk apa yang paling laris Jumat kemarin?") dan mengeksekusi multi-step reasoning terhadap data nyata di MongoDB Atlas, lalu memberikan aksi konkret, bukan sekadar jawaban.

## 6. Agent Intent Classification

Agent harus bisa handle minimal 8 intent ini. Setiap intent punya multi-step execution:

| Intent | Contoh Query | Multi-Step Execution |
|---|---|---|
| `check_stock` | "Stok mie instan berapa?" | 1. Parse produk → 2. Query Atlas → 3. Compare with minimum → 4. Return status + alert jika low |
| `check_debt` | "Hutang Bu Sari berberapa?" | 1. Resolve customer name → 2. Query debt_history → 3. Calculate total → 4. List detail per item |
| `record_sale` | "Tadi jual 3 aqua sama 2 rokok sampoerna" | 1. Parse items + qty → 2. Lookup product_id → 3. Validate stock cukup → 4. Insert transaction → 5. Update stock → 6. Return receipt |
| `sales_report` | "Berapa pendapatan hari ini?" | 1. Set date range → 2. Aggregate transactions → 3. Group by category → 4. Format laporan |
| `restock_alert` | "Produk apa yang mau habis?" | 1. Query semua produk → 2. Compare stock_current vs stock_minimum → 3. Rank by urgency → 4. Return prioritized list |
| `debt_collection` | "Siapa yang belum bayar hutang minggu ini?" | 1. Query customers → 2. Filter debt_total > 0 → 3. Filter by date → 4. Sort by amount → 5. Return list |
| `record_payment` | "Bu Sari bayar hutang 50 ribu" | 1. Resolve customer → 2. Update debt_history → 3. Update debt_total → 4. Insert transaction → 5. Confirm |
| `sales_forecast` | "Kira-kira besok bakal ramai?" | 1. Query 30-day history → 2. Day-of-week pattern → 3. Calculate avg → 4. Return prediction + top items |

## 6A. Intent → MCP Tool → Collection Mapping (DECIDED)

> **Gap resolved:** Setiap intent punya mapping eksplisit ke MCP tool dan collection. Agent Builder memanggil tool ini berurutan; tidak ada improvisasi saat implementasi.

### MCP Tools yang Dipakai

| MCP Tool | Fungsi | Dipakai oleh |
|---|---|---|
| `find` / `query` | Baca dokumen dengan filter | Semua read intent |
| `aggregate` | Pipeline aggregation Atlas | `sales_report`, `sales_forecast`, `debt_collection` |
| `insertOne` | Insert dokumen baru | `record_sale`, `record_payment` |
| `updateOne` | Update dokumen existing | `record_sale` (stok), `record_payment` (hutang) |

> Nama tool mengikuti `@modelcontextprotocol/server-mongodb`. Jika wrapper Agent Builder beda naming, map 1:1 ke operasi di atas — jangan ganti logika.

### Mapping Lengkap per Intent

| Intent | Tier | MCP Tools (urutan) | Collection | Write? |
|---|---|---|---|---|
| `check_stock` | 1 | `find` → (compare in agent) | `products` | ❌ |
| `check_debt` | 1 | `find` | `customers` | ❌ |
| `restock_alert` | 1 | `find` `{stock_current: {$lte: $expr: "$stock_minimum"}}` → sort | `products` | ❌ |
| `sales_report` | 1 | `aggregate` (match date → group by category → sum) | `transactions` | ❌ |
| `record_sale` | 1 | `find` products → **konfirmasi user** → `insertOne` transactions → `updateOne` products (×N) | `products`, `transactions` | ✅ |
| `record_payment` | 1 | `find` customers → **konfirmasi user** → `updateOne` customers → `insertOne` transactions | `customers`, `transactions` | ✅ |
| `debt_collection` | 2 | `find` `{debt_total: {$gt: 0}}` → filter date → sort | `customers` | ❌ |
| `sales_forecast` | 2 | `aggregate` (30-day → group by dayOfWeek → avg) | `transactions` | ❌ |

### Action Engine (bukan intent terpisah)

| Action | Trigger | Sumber data |
|---|---|---|
| `restock_alert` | Auto setelah `check_stock` jika low | Hasil query `products` |
| `debt_summary` | Auto setelah `check_debt` | Hasil query `customers` |
| `sales_forecast` | Intent `sales_forecast` | Aggregation `transactions` |
| `promo_suggestion` | Stretch — setelah `sales_report` weekly | Produk qty < 20% rata-rata minggu lalu |

### Write Flow Wajib (Atomic Intent)

```
record_sale:
  1. find products (resolve SKU + validate stock)
  2. Jika ambigu → STOP, minta konfirmasi (jangan write)
  3. insertOne → transactions {type: "sale", items[], total, ...}
  4. updateOne → products {$inc: {stock_current: -qty}} per item
  5. Return receipt + restock_alert jika perlu

record_payment:
  1. find customers (resolve nama)
  2. Jika ambigu → STOP, minta konfirmasi
  3. updateOne → customers {debt_total, debt_history.paid}
  4. insertOne → transactions {type: "sale", payment_method: "tunai", ...}
  5. Return sisa hutang
```

## 6B. Strategi Fuzzy Matching (DECIDED)

> **Gap resolved:** Pipeline resolusi nama produk dan customer sudah ditentukan. Jangan pilih strategi lain saat coding.

### Resolusi Nama Produk (4 langkah berurutan)

```
Input user → Normalisasi → Langkah 1 → 2 → 3 → 4 → Hasil
```

**Langkah 0 — Normalisasi input**
- Lowercase, trim whitespace
- Hapus filler: "yang", "deh", "dong", "bu", "pak", "tolong"
- Normalisasi angka: "dua" → 2, "limapuluh ribu" → 50000

**Langkah 1 — Exact alias match** (paling cepat, paling akurat)
```javascript
// Query via MCP find
{
  $or: [
    { name: { $regex: "^mie instan$", $options: "i" } },
    { name_aliases: { $regex: "^mie instan$", $options: "i" } }
  ]
}
```
- 1 match → pakai langsung
- 0 match → lanjut Langkah 2

**Langkah 2 — Partial text match** (Atlas Search atau regex)
```javascript
{
  $or: [
    { name: { $regex: "mie instan", $options: "i" } },
    { name_aliases: { $regex: "mie instan", $options: "i" } }
  ]
}
```
- Score: exact substring match = 1.0, prefix match = 0.8, contains = 0.6
- 1 match dengan score ≥ 0.8 → pakai langsung
- Multiple match atau score 0.6–0.8 → lanjut Langkah 3

**Langkah 3 — Vector search** (fallback untuk typo / variasi ejaan)
- Embed query user via **Gemini embedding API** (768 dimensions — sesuai vector index)
- `$vectorSearch` pada index `products_vector_index`, field `name`
- Top 1 dengan score ≥ 0.85 → pakai langsung
- Top 2+ dengan score gap < 0.15 → **disambiguasi** (Langkah 4)

**Langkah 4 — Disambiguasi ke user**
```
Agent: "Maksudnya yang mana, Bu?
  1. Indomie Goreng Original (stok: 24)
  2. Indomie Goreng Ayam Bawang (stok: 12)
  3. Mie Sedap Goreng (stok: 8)"
```
- Max 3 opsi, urut by score
- User pilih → lanjutkan intent
- Tidak ada match sama sekali → suggest 3 terdekat dari Langkah 3, **jangan auto-insert**

### Resolusi Nama Customer (lebih sederhana)

```
1. Normalisasi: hapus "bu", "pak", "mbak" → sisakan nama inti
2. find customers { name: { $regex: "sari", $options: "i" } }
3. 1 match → pakai | 2+ match → disambiguasi | 0 match → tanya buat customer baru
```

### Threshold Ringkas

| Kondisi | Aksi |
|---|---|
| 1 match, score ≥ 0.8 | Auto-resolve |
| 2+ match, score gap < 0.15 | Disambiguasi ke user |
| 0 match setelah Langkah 3 | Suggest similar, no write |
| Write intent + ambigu | **WAJIB konfirmasi** sebelum MCP insert/update |

```

  - Generate session_id (crypto.randomUUID()) saat first visit
  - Simpan di localStorage key "wargio_session_id"
  - Kirim di header: X-Session-Id: <uuid>

Backend FastAPI:
  - Baca X-Session-Id, fallback generate baru jika kosong
  - Load/create agent_sessions document by session_id
  - Context chat disimpan per session — tidak cross-contaminate

Atlas:
  - Satu cluster, satu database "wargio_demo"
  - Semua collection shared (single warung)
  - Seed data: 1 warung fiktif "Warung Bu Sari"
```

```
You are Wargio, an AI business assistant for Indonesian micro-retailers (warung owners).

CAPABILITIES:
- Check and report stock levels with restock recommendations
- Record sales transactions (multi-item, with or without customer name)
- Track and manage customer debts (catat hutang, catat pembayaran)

- Identify best-selling and slow-moving products
- Forecast demand based on historical patterns

RULES — ABSOLUTE:
1. ALWAYS verify data from Atlas before stating any number
2. ALWAYS ask for confirmation before writing to the database (insert/update)
3. NEVER assume product names — use fuzzy matching pipeline (Section 6B), then confirm if ambiguous
4. If a customer name matches multiple records, show options and ask which one
5. If stock is insufficient for a sale, REJECT and inform the user clearly
6. All currency in Indonesian Rupiah (Rp), formatted with dots (Rp 12.500)
7. Dates in Indonesian format (Senin, 2 Juni 2026)

MULTI-STEP EXECUTION:
When you receive a complex request, think step by step:
1. Identify the intent
2. Extract entities (product names, customer names, quantities, dates)
3. Resolve ambiguities (ask if needed)
4. Execute MCP tool calls in sequence
5. Validate results
6. Respond in natural, friendly Bahasa Indonesia

ERROR HANDLING:
- MCP tool error → tell user "Ada gangguan teknis, coba lagi sebentar" and retry once
- Product not found → suggest similar products from database
```

#### MongoDB Driver Decision (Gap Resolved)

> Motor sudah deprecated/EOL pada 2026. Untuk implementasi baru, gunakan **PyMongo Async API**.

- Pilihan utama: `pymongo` async (`AsyncMongoClient`)
- Posisi di dokumen lama yang menyebut Motor dianggap legacy reference
- Aturan final: driver async wajib non-blocking dan terhubung Atlas live

### 16C. Frontend (Web)

- Framework: Next.js **App Router** (line terbaru 2026)
- Bahasa: TypeScript strict (`"strict": true`)
- Styling: Tailwind CSS
- UI components: shadcn/ui
- Prinsip rendering:
  - Server Components default
  - `use client` hanya untuk komponen interaktif

### 16D. Testing Stack

- Backend: `pytest`, `pytest-asyncio`, `httpx` (API test)
- Frontend: `vitest`, `@testing-library/react`, `@testing-library/jest-dom`
